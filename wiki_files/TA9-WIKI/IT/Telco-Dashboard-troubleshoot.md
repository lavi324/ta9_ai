If after cloning Isuzu env the person dashboard drop next errors 
![image.png](/.attachments/image-ab3cfc35-a686-46d0-94db-1dc98d176004.png)

The solution is to check the person dashboard in the next path: c:\Windows\System32

in System32 contains some JSON files that belong to the env. 
![image.png](/.attachments/image-143d026b-016b-4a7a-b32a-1a410e775525.png)

Edit the files to correct windows and Linux IPs and reset the argus host. after reset, the system should recognize the new IPs
