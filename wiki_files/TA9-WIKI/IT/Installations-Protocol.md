**Pre - Requisites:**
**1. Set a sync meeting** 
1-2 weeks before the actual upgrade day with the relevant team members including: 
- VP R&D
- QA
- IT manager
- CTO
- Developer 
- PM


**2. Prepare the client**
- Set an installation day and shutdown time (important)
- Alert the client that a shut-down is required for 1 full day at least
- Ask the client to make sure to have a valid backup a day before
- For an on-prem installation - Ask the client to prepare 2 workstations for TA9 Team 

*set a calendar reminder for yourself to remind the client a day before the due day
 
**3.Contacts**
Provide the Installation team with client-side contacts:
- PM
- IT manager

**4. File Scan**
If a file scan is needed, notify the client that TA9 software weighs approx 3GB.
Estimate how long it would take to scan those and make sure you have enough time for scanning prior to the installation day

**5. On installation Day**
- Notify all users on the shutdown, Make sure it is consistent with the time of day agreed upon 
- Make sure there's a valid snapshot taken by the Client IT Team
- Follow the Shutdown protocol, print it first in cases that web access isn't available
- Install, and perform a sanity test according to the env'

**6. Documentation**
- Keep track of documentation, passwords & Ip's managed
- Make sure to document the environment components, servers, versions, configurations and other details that may come in handy on the local TA9 environment.

**7. Finish up**
After installation is done: 
- IT to preform Sanity. Make sure everything works and configured
- Notify the Client's IT team, on-prem about the completion
- Notify TA9 PM 
- On TA9 PM responsibility to notify the client's PM's and management + Sent a release note if required

**8. QA**
Book QA tester for as long as needed, (to visit Site after installation)


**9. Alerting / Rollback** (For TA9 IT)
- Alert TA9 PM on installation progress during the day when possible
- Alert TA9 PM on every issue, blockage or problems that occurs
- Make sure to Keep PM updated with changes, special requests, or any additional update about the installation status
- In case there are problems completing the installation, perform a "rollback" from the backup
and alert the client & PM's by sending a summarizing email.
