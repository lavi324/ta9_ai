# WildFly

## Redeploy services

For all services such as : 

BlotterService.war
EntitiesServices.war
FaceRecognition.war
FreeTextService.war
MediaAnalyzingService.war
MessageBroker.war
ReportsGenerationService.war
TextAnalyticsService.war

This file are deployed to our system needs to serve requests from the our Web interface.

From time to time this services are get stuck, to redeploy theme find the file for exemple:

FaceRecognition.war.deployed

And delete it. 

It becomes to FaceRecognition.war.undeploying

Once the status changed to:
FaceRecognition.war.undeployed

Restart wildfly by SSH to the machine, write the command : 

`sudo systemctl restart wildfly`

![image.png](/.attachments/image-6d92209b-e200-4f80-aebf-c31c4d8631fe.png)

To see the status of the wildfly service: 

`sudo systemctl status wildfly`

![image.png](/.attachments/image-ef3b1da0-4626-45c7-8134-a29545ea1368.png)

Once the service restarted, delete FaceRecognition.war.undeployed

You will see that the file is FaceRecognition.war.dodeploy

After few seconds it become to :

FaceRecognition.war.deployed.

Done!
